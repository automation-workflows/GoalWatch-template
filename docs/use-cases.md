# GoalWatch Use Cases

## 1. Custom event alerts

A client defines their own condition in plain language:

- "Alert me when this wallet ratio goes above 2.5"
- "Tell me when this dashboard count crosses 100"
- "Notify me when this page says the campaign is live"

This is the core GoalWatch use case.

## 2. Web3 monitoring

A Web3 site can expose a status page, dashboard, or metric page. GoalWatch watches it and notifies the user when the target condition is met.

Examples:

- token price threshold
- liquidity ratio threshold
- campaign status change
- mint / sale / listing live
- community count milestone

## 3. Finance and market tracking

- asset price targets
- moving-average or percentage alerts on public dashboards
- exchange listing announcements
- portfolio condition triggers

## 4. Sports and live event tracking

- score thresholds
- team totals
- live event status pages
- match outcome milestones

## 5. Product and campaign tracking

- landing page goes live
- form submissions reach a count
- waitlist crosses a target
- feature flag or release status changes

## 6. Simple "continue receiving alerts" button

A page can expose a clear CTA like:

- "Continue receiving alerts"
- "Add bot to Telegram"
- "Message bot on Discord"
- "Confirm test notification"

That makes the product feel like a lightweight alert onboarding step instead of a full dashboard.

In the MVP, the user can also enter the target URL manually. Later, that target can be preconfigured by the developer in a secret or site setting so the UI only keeps the goal and notification steps.

## Recommended MVP positioning

Describe GoalWatch as:

> "Set a goal in plain language, choose Telegram or Discord, and get notified when it happens."

That keeps the product simple and clear for first-time users.
